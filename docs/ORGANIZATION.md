## Qué contratar primero (plan de delegación)

No porque vayas a contratar mañana — para **saber qué soltar primero** cuando el negocio lo pida.

| Orden | Rol | Cuándo | Por qué |
|-------|-----|--------|---------|
| **1** | **Full Stack Developer** | Cuando MRR o pilotos justifiquen ~40 h/mes de dev | Libera ~60 % del trabajo técnico (Core + Hub + fixes prod) |
| **2** | **Customer Success / Soporte** | Cuando lleguen clientes activos | Protege tu tiempo; evita que soporte coma desarrollo |
| **3** | **Marketing** | Después de producto vendible | Mensaje y funnel; no antes de Core listo |
| **4** | **Sales** | Cuando el producto cierre solo en demo | Tú dejas de ser único vendedor |
| **5** | **DevOps** | Cuando infra supere ~5 h/semana o incidentes | Hasta entonces: Railway + scripts + smokes bastan |
